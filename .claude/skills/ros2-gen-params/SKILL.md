# ROS2 Parameter File Generator

Generate a parameter YAML file for a ROS2 node. Arguments: `$ARGUMENTS`

Parse arguments:
- **node**: first argument — node name (e.g. `/move_base`). Prepend `/` if missing.
- **--output file**: write to this file (default: `<node_name>_params.yaml`)
- **--template**: generate a template with all params set to their default/current values and comments
- **--namespace ns**: wrap the output under this ROS2 namespace

## Steps

1. **Verify the node is running**:
   ```bash
   ros2 node list
   ```
   If not found, suggest checking the node name and stop.

2. **List all parameters**:
   ```bash
   ros2 param list <node>
   ```

3. **Dump all current parameter values**:
   ```bash
   ros2 param dump <node>
   ```
   This produces a YAML block like:
   ```yaml
   /move_base:
     ros__parameters:
       base_global_planner: navfn/NavfnROS
       base_local_planner: base_local_planner/TrajectoryPlannerROS
       clearing_rotation_allowed: true
       controller_frequency: 5.0
   ```

4. **Get parameter types and descriptions** for each parameter:
   ```bash
   ros2 param describe <node> <param_name>
   ```
   Collect: type, description, constraints (min/max/step), read-only flag.

5. **Build the output YAML**:

   **Standard format** (without --template):
   ```yaml
   # Auto-generated by /ros2-gen-params
   # Node: /move_base
   # Generated: 2026-03-08

   /move_base:
     ros__parameters:
       base_global_planner: navfn/NavfnROS
       controller_frequency: 5.0
       clearing_rotation_allowed: true
   ```

   **Template format** (with --template, adds type annotations and comments):
   ```yaml
   # Parameter file for /move_base
   # Generated: 2026-03-08
   # Edit values below, then load with:
   #   ros2 param load /move_base <this_file>
   # Or at launch with:
   #   parameters=[<this_file>]

   /move_base:
     ros__parameters:

       # Global planner plugin name
       # Type: string
       base_global_planner: "navfn/NavfnROS"

       # Control loop frequency (Hz)
       # Type: double  |  Range: 1.0 – 50.0
       controller_frequency: 5.0

       # Allow in-place rotation during clearing
       # Type: bool
       clearing_rotation_allowed: true
   ```

6. **If --namespace flag**, wrap under that namespace:
   ```yaml
   /my_robot:
     /move_base:
       ros__parameters:
         controller_frequency: 5.0
   ```

7. **Determine output filename**:
   - If --output given, use that path
   - Otherwise: strip leading `/` from node name, replace `/` with `_`, append `_params.yaml`
     e.g. `/move_base` → `move_base_params.yaml`

8. **Write the file** and report:
   ```
   ✅ Written: move_base_params.yaml
      Parameters: 23
      Node:       /move_base

   Load at runtime:
     ros2 param load /move_base move_base_params.yaml

   Load at launch (Python):
     Node(
       package='my_pkg',
       executable='move_base',
       parameters=['move_base_params.yaml'],
     )
   ```

9. **If the node is NOT running** and --template is given, generate a minimal template stub instead:
   ```yaml
   # Template for <node_name>
   # Fill in your parameter values below

   /<node_name>:
     ros__parameters:
       # Add parameters here
       example_param: value
   ```
   Suggest running the node first to capture real defaults.
